Seleccione un archivo de reglas de carga para especificar cómo cargar el espacio de trabajo de compilación. Si las reglas de carga dinámica se han especificado utilizando una extensión, estas reglas tienen prioridad sobre las que se han especificado aquí.

Para aplicar reglas de carga, lístelas en el formato JSON siguiente, guarde la lista en un archivo en disco y especifique la vía de acceso del archivo en este campo. 

  {
    "loadRules":[
      {
        "componentName":"JUnit",
        "filePath":"JUnit Project/junit-project.loadrule"
      },
      {
        "componentName":"Releng",
        "filePath":"Releng Project/releng-project.loadrule"
      }
    ]
  }

También puede especificar "componentId" en lugar de "componentName." El "componentId" es el ID del elemento ID (UUID) del componente. Si los especifica ambos, "componentName" y "componentId," solo se produce la búsqueda por "componentId."

El valor "filePath" es la vía de acceso completa del archivo de reglas de carga en el repositorio; es decir, la vía de acceso remota. También puede especificar "fileItemId" en lugar de "filePath." El "fileItemId" es el ID del elemento (UUID) del archivo de reglas de carga. Al especificar la vía de acceso del archivo, utilice barras diagonales (/) para separar los segmentos de la vía de acceso. Si especifica ambos, "filePath" y "fileItemId," solo se produce la búsqueda por "fileItemId."
