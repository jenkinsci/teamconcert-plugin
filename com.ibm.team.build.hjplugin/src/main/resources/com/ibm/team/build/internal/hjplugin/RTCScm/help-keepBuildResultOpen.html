This feature is useful when implementing a delivery pipeline within Jenkins. When checked, Jenkins will not close the RTC Build Result on the server side, allowing subsequent jobs to write content into it and change its result.

It should be used in conjunction with the boolean property com.ibm.rational.connector.hudson.queueOnly on the Build Engine, so that builds triggered from the RTC Client also remain open after the respective Jenkins build has finished.

Notice that a mechanism to close the RTC Build Result in afterwards should be provided. See Ant Tasks.
