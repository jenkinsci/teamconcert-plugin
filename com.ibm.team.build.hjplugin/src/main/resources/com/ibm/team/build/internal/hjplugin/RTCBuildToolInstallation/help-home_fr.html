Chemin d'accès aux outils de génération. Il s'agit du répertoire dans lequel sont installés les outils de génération RTC. Exemple : C:\Program Files (x86)\IBM\TeamConcertBuild\buildsystem\buildtoolkit

Le chemin d'accès mène aux outils de génération sur le noeud maître. Si vous disposez d'une configuration de génération répartie (noeuds esclaves), vous devez aussi spécifier l'emplacement des outils de génération sur le noeud esclave (si le chemin d'accès est différent). Après la sauvegarde, accédez à la configuration du noeud esclave. Dans la section des propriétés de noeud, ajoutez les emplacements de tous les outils de génération à utiliser sur le noeud esclave. Voir Master/Slave Configuration pour des détails.

Si vous prévoyez de charger des liens symboliques, des étapes de configuration supplémentaires sont nécessaires. L'accès aux bibliothèques natives est requis pour la création de liens symboliques. Pour plus de détails sur la configuration, voir Team Concert Plugin
