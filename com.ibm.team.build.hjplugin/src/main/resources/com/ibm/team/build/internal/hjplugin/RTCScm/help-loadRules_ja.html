ビルド・ワークスペースのロード方法を指定するロード規則ファイルを選択します。拡張子を使用して動的ロード規則が指定された場合は、ここに指定された規則よりそれらの規則の方が優先されます。

ロード規則を適用するには、それらを JSON フォーマットでリストし、そのリストをディスク上のファイルに保存し、そのファイルのパスをこのフィールドに指定してください。 

  {
    "loadRules":[
      {
        "componentName":"JUnit",
        "filePath":"JUnit Project/junit-project.loadrule"
      },
      {
        "componentName":"Releng",
        "filePath":"Releng Project/releng-project.loadrule"
      }
    ]
  }

「componentName」の代わりに「componentId」を指定することもできます。「componentId」はコンポーネントの項目 ID (UUID) です。「componentName」と「componentId」の両方を指定した場合、ルックアップは「componentId」によってのみ行われます。

「filePath」の値は、リポジトリー内のロード規則ファイルの絶対パス、つまりリモート・パスです。 また、「filePath」の代わりに「fileItemId」を指定することもできます。「fileItemId」はロード規則ファイルの項目 ID (UUID) です。ファイル・パスを指定する場合は、スラッシュ (/) を使用してパス・セグメントを区切ってください。「filePath」と「fileItemId」の両方を指定した場合、ルックアップは「fileItemId」によってのみ行われます。
