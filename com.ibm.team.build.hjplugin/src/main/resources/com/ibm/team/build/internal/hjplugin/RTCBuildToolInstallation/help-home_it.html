Il percorso del toolkit della build. Questa sarà una directory compresa nei tool di build RTC installati. Esempio: C:\Program Files (x86)\IBM\TeamConcertBuild\buildsystem\buildtoolkit

Il percorso è quello del toolkit della build in quello principale. Se si dispone di una configurazione di build distribuita (secondarie), sarà inoltre necessario specificare dove si trova il toolkit della build in quello secondario (supponendo che sia in un percorso differente). Dopo aver salvato, andare sulla configurazione in quello secondario. Nella sezione delle proprietà del nodo, aggiungere le ubicazioni degli strumenti per ogni toolkit della build da utilizzare in quello secondario. Per i dettagli, consultare Master/Slave Configuration.

Se si intende caricare link simbolici, è necessario eseguire alcune configurazioni aggiuntive. L'accesso alle librerie native è richiesto per creare link simbolici. Per ulteriori dettagli sulla configurazione, visitare Team Concert Plugin
