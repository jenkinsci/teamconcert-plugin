빌드 작업공간을 로드하는 방법을 지정하는 로드 규칙 파일을 선택하십시오. 확장자를 사용하여 동적 로드 규칙이 지정된 경우 이러한 규칙은 여기 지정된 규칙보다 우선합니다.

로그 규칙을 적용하려면 다음 JSON 형식으로 해당 규칙을 목록화하고, 이 목록을 디스크에 있는 파일로 저장하고, 이 필드에 해당 파일에 대한 경로를 지정하십시오. 

  {
    "loadRules":[
      {
        "componentName":"JUnit",
        "filePath":"JUnit Project/junit-project.loadrule"
      },
      {
        "componentName":"Releng",
        "filePath":"Releng Project/releng-project.loadrule"
      }
    ]
  }

"componentName" 대신 "componentId"를 지정할 수도 있습니다. "componentId"는 컴포넌트의 항목 ID(UUID)입니다. "componentName"과 "componentId"를 둘 다 지정하는 경우 "componentId"로만 검색이 수행됩니다.

"filePath" 값은 저장소의 로그 규칙 파일에 대한 전체 경로입니다. 즉, 원격 경로입니다. "filePath" 대신 "fileItemId"를 지정할 수도 있습니다. "fileItemId"는 로그 규칙 파일의 항목 ID (UUID)입니다. 파일 경로를 지정하는 경우 슬래시(/)를 사용하여 경로 세그먼트를 분리하십시오. "filePath"와 "fileItemId"을 둘 다 지정하는 경우 "fileItemId"로만 검색이 수행됩니다.
