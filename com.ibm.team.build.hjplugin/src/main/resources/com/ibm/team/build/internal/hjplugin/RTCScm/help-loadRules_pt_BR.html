Selecione um arquivo de regras de carregamento para especificar como carregar a área de trabalho de compilação. Se as Regras de Carregamento Dinâmico foram especificadas usando uma extensão, essas regras terão precedência sobre as especificadas aqui.

Para aplicar regras de carregamento, liste-as no formato JSON a seguir, salve a lista em um arquivo no disco e especifique o caminho para o arquivo neste campo. 

  {
    "loadRules":[
      {
        "componentName":"JUnit",
        "filePath":"JUnit Project/junit-project.loadrule"
      },
      {
        "componentName":"Releng",
        "filePath":"Releng Project/releng-project.loadrule"
      }
    ]
  }

Também é possível especificar o "componentId" em vez de "componentName". O "componentId" é o ID do item (UUID) do componente. Se você especificar "componentName" e "componentId," a consulta ocorrerá somente por "componentId".

O valor de "filePath" é o caminho completo para o arquivo de regra de carregamento no repositório; ou seja, o caminho remoto. Também é possível especificar o "fileItemId" em vez de "filePath". O "fileItemId" é o ID do item (UUID) do arquivo de regra de carregamento. Ao especificar o caminho de arquivo, use barras (/) para separar os segmentos do caminho. Se você especificar "filePath" e "fileItemId", a consulta ocorrerá somente por "fileItemId".
