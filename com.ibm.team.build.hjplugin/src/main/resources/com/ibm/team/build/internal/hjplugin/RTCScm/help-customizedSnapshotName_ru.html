Задает имя моментальной копии, создаваемой после принятия изменений.

В этом поле можно указать любую пользовательскую строку с ссылками на переменные среды и параметры компоновки. Рекомендуется добавить по крайней мере одну переменную с уникальным значением для каждой компоновки (например, BUILD_NUMBER, BUILD_ID или BUILD_URL), чтобы последовательные компоновки создавали моментальные копии с разными именами. Если это поле пустое, то применяется имя моментальной копии по умолчанию. Для заданий на основе определения компоновки по умолчанию применяется имя моментальной копии "${buildDefinitionId}_#${BUILD_NUMBER}". Для заданий на основе потока или рабочей области хранилища по умолчанию применяется имя моментальной копии "${JOB_NAME}_#${BUILD_NUMBER}".

Например, если заданию присвоено имя "JUnit Integration Build" и это поле содержит значение "${JOB_NAME}_#${BUILD_NUMBER}_Snapshot", то имя в ходе первой компоновки будет создана моментальная копия с именем "JUnit Integration Build_#1_Snapshot".
